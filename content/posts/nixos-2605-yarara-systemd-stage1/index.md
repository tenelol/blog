## 背景

2026年5月30日、NixOS 26.05 "Yarara" が正式にリリースされた。2842人のコントリビュータによる59,703コミットを経た今回のリリースは、単なるパッケージ更新にとどまらず、ブートプロセスの根幹に関わる変更が含まれている。

NixOS は「宣言的な設定」と「再現性」を軸にした Linux ディストリビューションで、サーバー管理や開発環境の構築で採用が増えている。Nix パッケージマネージャ単体であれば macOS や他の Linux ディストロでも利用でき、開発者ツールとしての裾野も広い。

今回のリリースで特に注目すべきは、stage 1（initrd）が systemd ベースにデフォルト切り替えされた点と、新しく導入された「switch inhibitors」の仕組みだ。既存ユーザーにとってはアップグレード時に確認すべき項目が多く、これから NixOS を触る人にとっても設計思想を理解するよい機会になる。

## systemd stage 1 がデフォルトに

Stage 1 とは、カーネルが読み込まれた直後に動く初期環境（initrd）のことで、ルートファイルシステムのマウントや暗号化ボリュームの解除といった処理を担当する。これまで NixOS は独自のシェルスクリプトでこの処理を行っていたが、26.05 から systemd ベースの実装がデフォルトになった。

従来のスクリプト実装は 26.11 で削除予定とされているため、カスタムの initrd スクリプトを使っている場合は移行計画を立てる必要がある。

systemd stage 1 への移行で得られるメリットは明確で、ブートプロセスが systemd のユニット管理に統一されることで、依存関係の解決やログ出力が標準化される。デバッグ時に `journalctl` で initrd のログを追えるようになるのは、トラブルシューティングの効率を大きく改善する。

```nix
# もし従来のスクリプト実装を維持したい場合（非推奨）
boot.initrd.systemd.enable = false;
```

ただし、この設定は 26.11 で使えなくなる予定なので、あくまで一時的な回避策として考えてほしい。

## switch inhibitors という新しい安全装置

26.05 で新たに導入された switch inhibitors は、世代の切り替え時に「本当に切り替えて大丈夫か」を事前チェックする仕組みだ。

NixOS では `nixos-rebuild switch` で新しいシステム構成に切り替えるが、たとえば systemd のバージョンが変わった場合、ライブシステムでそのまま切り替えると予期しない挙動が起きる可能性がある。switch inhibitors は、前の世代と新しい世代の間で指定された値を比較し、差分がある場合に切り替えを拒否する。

```nix
# systemd のバージョンが変わったら切り替えを止める例
system.switchInhibitors = [
  config.systemd.package.version
];
```

実際、今回のリリースでは D-Bus 実装が `dbus` から `dbus-broker` に切り替わっており、これ自体が switch inhibitor として設定されている。強制的に切り替えたい場合は `NIXOS_NO_CHECK=1` を設定すれば通るが、本番サーバーでは再起動での切り替えが安全だ。

## x86_64-darwin サポートの終了予告

macOS ユーザーにとって見逃せないのが、x86_64-darwin（Intel Mac）のサポート終了予告だ。26.05 が x86_64-darwin をサポートする最後のリリースとなり、2026年末のサポート期限をもってバイナリのビルドも停止される。

Apple が Intel Mac の販売を終了してから時間が経ち、ビルドインフラの維持コストと開発者リソースを考慮した判断とのこと。Intel Mac で Nix を使っている場合は、aarch64-darwin（Apple Silicon）への移行を計画する必要がある。

## その他の注目アップデート

パッケージ面では、20,442の新規パッケージが追加され、17,532の古いパッケージが削除された。積極的な整理によってパッケージセットの健全性が保たれている。

デスクトップ環境では GNOME 50 "Tokyo" が搭載され、アクセシビリティの改善とディスプレイ管理の強化が行われた。コンパイラは GCC 15 に更新、LLVM は 21 のまま据え置きとなっている。

モジュール面では85の新規モジュールと1,547の設定オプションが追加された一方、25モジュールと355オプションが削除されており、NixOS 特有の「使わないものは消す」姿勢が一貫している。

## 気をつけるポイント

アップグレードの際にまず確認すべきは、カスタムの initrd スクリプトの有無だ。systemd stage 1 への移行は段階的に進められてきたが、デフォルトが切り替わったことで、これまで暗黙的にスクリプト実装を使っていた構成が影響を受ける可能性がある。

また、activation script からの systemd ユニット操作（reload / restart）も非推奨になっている。26.11 で削除予定なので、該当する処理がある場合は早めに書き換えを検討したい。

アップグレード自体は公式ドキュメントの手順に沿って `nix-channel --update` と `nixos-rebuild switch` で実行できるが、switch inhibitors が作動する場合は再起動が必要になることを覚えておくとよい。

## 今後の展望

次のリリースは NixOS 26.11 "Zokor" で、スクリプトベースの stage 1 と一部の activation script が完全に削除される見込みだ。NixOS は「宣言的であること」の純度を着実に高めており、systemd への統一はその流れの中核にある。再現性のある Linux 環境に興味があるなら、26.05 は試してみる価値のあるリリースだ。
